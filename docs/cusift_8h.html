This is a convenience function that combines ExtractSiftFromImage(), MatchSiftData(), FindHomography(), and WarpImages() into a single call. This function is useful for applications that require both feature matching and image alignment, such as panorama stitching or object recognition. It provides a streamlined interface for performing the entire workflow with a single function call.

Given the computed homography, warp the input images to align them. The warped images are returned in the 'warped_image1' and 'warped_image2' output parameters. The caller is responsible for ensuring that the input images and homography are valid before calling this function, and for freeing any resources associated with the warped images when done. To free the warped images call the cstdlib free() function on the 'host_img_' field of the Image_t structures, and set the pointer to nullptr to avoid dangling pointers. We use malloc to allocate space for the warped images.

Parameters
image1Pointer to the first input image.
image2Pointer to the second input image.
homographyPointer to a 3x3 matrix in row-major order representing the homography transformation.
warped_image1Pointer to the Image_t structure where the warped first image will be stored.
warped_image2Pointer to the Image_t structure where the warped second image will be stored.
useGPUBoolean flag indicating whether to use GPU acceleration for the warping operation.
